# 1. Objetivo

Integrar las correcciones solares ya calculadas para obtener la longitud
aparente del Sol.

Este algoritmo no introduce matemática nueva. Solo combina magnitudes físicas
producidas por algoritmos anteriores.

---

# 2. Entrada

AST-0007 recibe:

```text
λg
```

longitud geocéntrica del Sol;

```text
aberración
```

corrección de aberración solar;

```text
Δψ
```

nutación en longitud.

---

# 3. Salida

La salida es:

```rust
Degrees
```

normalizado al intervalo:

```text
[0°, 360°)
```

---

# 4. Fórmula Implementada

```text
λ☉ = normalize_360(λg + aberración + Δψ)
```

No se calcula ninguna de las entradas dentro de este algoritmo.

---

# 5. API Pública

```rust
pub fn apparent_solar_longitude(
    geocentric_solar_longitude: Degrees,
    solar_aberration: Degrees,
    nutation_in_longitude: Degrees,
) -> Degrees;
```

---

# 6. Tests

Casos cubiertos:

- suma simple de `λg`, aberración y `Δψ`;
- correcciones nulas;
- normalización por encima de `360°`;
- normalización por debajo de `0°`;
- signos independientes para aberración y nutación.

---

# 7. Numerical Notes

- Se usa `f64`.
- La función no calcula VSOP87D.
- La función no calcula radio vector.
- La función no calcula aberración.
- La función no calcula nutación.
- La normalización ocurre una sola vez al final.
