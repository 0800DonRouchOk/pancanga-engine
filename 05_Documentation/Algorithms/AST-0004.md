# 1. Objetivo

Calcular el radio vector heliocéntrico de la Tierra usando `VSOP87D`.

Este valor representa la distancia Tierra-Sol en unidades astronómicas y será
necesario para calcular la aberración solar en `AST-0005`.

---

# 2. Entrada

La entrada pública del algoritmo es:

```text
JulianDate
```

Internamente VSOP87D usa:

```text
tau = (JD - 2451545.0) / 365250.0
```

`tau` representa miles de años julianos desde J2000.0.

---

# 3. Salida

La salida es:

```rust
AstronomicalUnits
```

El valor corresponde al radio vector `R` publicado por VSOP87D para la Tierra.

---

# 4. Fórmula Implementada

```text
R(tau) = Σ tau^n * Σ A_i cos(B_i + C_i tau)
```

No se aplica división por `1e8` porque el archivo oficial `VSOP87D.ear` ya
publica las amplitudes de distancia en unidades astronómicas.

---

# 5. Coeficientes

La implementación usa los 997 términos del radio vector terrestre publicados en
`VSOP87D.ear`:

- `R0`: 526 términos.
- `R1`: 292 términos.
- `R2`: 139 términos.
- `R3`: 27 términos.
- `R4`: 10 términos.
- `R5`: 3 términos.

Total: 997 términos.

Los coeficientes fueron copiados con el mismo formato decimal visible en el
archivo oficial de IMCCE. No se truncaron manualmente ni se convirtieron a
`f32`.

---

# 6. API Pública

```rust
pub fn earth_radius_vector(jd: JulianDate) -> AstronomicalUnits;
```

# 7. Tests

Casos cubiertos:

- cálculo de `tau`;
- radio vector positivo;
- 10 valores oficiales de `vsop87.chk`;
- error máximo observado contra los valores oficiales.

---

# 8. Numerical Notes

- Se usa `f64`.
- No se usa `f32`.
- No se usa SIMD.
- No se fuerza FMA.
- No se truncaron coeficientes para optimizar tamaño o velocidad.
- La evaluación de `tau^n` se hace iterativamente, no con `powi`.
- La salida usa un tipo fuerte (`AstronomicalUnits`) para evitar mezclar
  distancias con ángulos o escalares genéricos.
