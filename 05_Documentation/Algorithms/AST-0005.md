# 1. Objetivo

Calcular y aplicar la corrección de aberración solar a la longitud geocéntrica
del Sol.

Este algoritmo introduce únicamente el fenómeno de aberración. No aplica
nutación, oblicuidad, ascensión recta ni reglas calendáricas.

---

# 2. Entrada

La corrección necesita:

```text
R
```

donde `R` es el radio vector Tierra-Sol en unidades astronómicas.

Para aplicar la corrección también se necesita:

```text
λg
```

donde `λg` es la longitud geocéntrica del Sol.

---

# 3. Salida

La corrección aislada es:

```rust
Degrees
```

La longitud corregida por aberración también es:

```rust
Degrees
```

---

# 4. Fórmula Implementada

Meeus da la corrección de aberración solar como:

```text
Δλ = -20.4898" / R
```

La implementación convierte esa corrección a grados:

```text
Δλ(deg) = (-20.4898 / R) / 3600
```

y luego aplica:

```text
λ = normalize_360(λg + Δλ)
```

# 6. Tests

Casos cubiertos:

- corrección para `R = 1 AU`;
- escalado inverso con `R`;
- aplicación sobre una longitud geocéntrica simple;
- normalización en el borde `0°`;
- verificación de que no se aplica nutación ni otra corrección.

---

# 7. Numerical Notes

- Se usa `f64`.
- La constante `20.4898"` se conserva explícitamente en segundos de arco.
- La conversión a grados ocurre una sola vez.
- La corrección se mantiene como valor angular firmado.
- La normalización se aplica solo después de sumar la corrección a `λg`.
- No se introduce infraestructura IAU 2000/2006 en esta etapa.
