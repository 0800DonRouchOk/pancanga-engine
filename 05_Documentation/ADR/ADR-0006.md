# ADR-0006: Modelo de Nutación en Longitud

Estado: Aceptado

Fecha: 11 de julio de 2026

## Contexto

La cadena solar de Pancanga Engine necesita incorporar la nutación en longitud
(`Δψ`) antes de producir la longitud aparente final del Sol.

El proyecto ya adoptó una línea de diseño coherente para la versión 1.0:

- `VSOP87D` para la posición heliocéntrica de la Tierra.
- Meeus para la corrección de aberración solar.
- Algoritmos pequeños, auditables y separados por fenómeno físico.

## Decisión

Pancanga Engine v1.0 adoptará la serie truncada de nutación de Jean Meeus para
calcular únicamente la nutación en longitud (`Δψ`).

La implementación de `AST-0006` no calculará oblicuidad ni nutación en
oblicuidad. Tampoco aplicará la corrección a la longitud solar; esa combinación
queda reservada para `AST-0007`.

## Justificación

- Mantiene coherencia con las decisiones solares anteriores.
- Tiene una implementación pequeña y fácil de auditar.
- Su precisión es suficiente para los objetivos de Pancanga Engine v1.0.
- Evita introducir prematuramente la infraestructura completa de IAU 2000A o
  IAU 2000B.
- La arquitectura permite sustituir el modelo por IAU 2000B o IAU 2000A en una
  versión futura sin modificar el resto del pipeline solar.

## Alternativas Consideradas

### IAU 2000A

Descartada para v1.0 por complejidad y tamaño. Puede ser una mejora futura si
el proyecto requiere precisión de efemérides más estricta.

### IAU 2000B

Descartada para v1.0 por la misma razón: aunque es más compacta que IAU 2000A,
agrega complejidad que no es necesaria para el objetivo actual.

### No aplicar nutación

Descartada. La longitud aparente del Sol requiere una corrección de nutación en
longitud para mantener la cadena física completa.

## Consecuencias

- `AST-0006` queda desbloqueado.
- `AST-0006` devuelve una corrección angular firmada (`Degrees`).
- `AST-0006` no normaliza la corrección.
- La oblicuidad queda fuera de alcance para este algoritmo.
- `AST-0007` combinará longitud geocéntrica, aberración y nutación.
