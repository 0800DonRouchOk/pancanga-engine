# ADR-0007: Referencia Externa de Validación Científica

Estado: Aceptado

Fecha: 11 de julio de 2026

## Contexto

Pancanga Engine busca construir un motor propio, offline, auditable y
suficientemente preciso para calcular Ekādaśī.

El proyecto no tiene como objetivo envolver Swiss Ephemeris ni delegar sus
cálculos principales a una biblioteca externa. Sin embargo, necesita una
referencia externa estable para validar empíricamente sus resultados.

Hasta ahora el Solar Engine v1.0 fue validado contra:

- datos publicados de IMCCE para VSOP87D;
- fórmulas de Jean Meeus;
- identidades geométricas y algebraicas del pipeline solar.

Antes de iniciar el modelo lunar conviene definir cuál será el oráculo externo
de comparación para el motor astronómico completo.

## Decisión

Pancanga Engine adoptará Swiss Ephemeris como referencia externa principal de
validación científica.

Cuando corresponda, también se considerarán los datos JPL que sustentan o
acompañan la validación de efemérides.

Esta decisión no cambia la fuente primaria de implementación de Pancanga
Engine. Meeus, VSOP87D y las decisiones ya aceptadas siguen siendo la base del
motor propio.

## Principio

Pancanga Engine no será un wrapper de Swiss Ephemeris.

Swiss Ephemeris funcionará como oráculo externo de validación:

```text
Pancanga Engine calcula offline con su propio código.
Swiss Ephemeris valida empíricamente los resultados.
```

## Consecuencias

- Cada subsistema astronómico importante deberá declarar si fue comparado contra
  Swiss Ephemeris o si esa comparación está pendiente.
- Las diferencias deberán medirse y documentarse con tolerancias explícitas.
- No se migrará un modelo astronómico solo por intuición o por preferencia
  teórica; primero se medirán las diferencias.
- Si las diferencias contra Swiss Ephemeris son pequeñas frente al objetivo de
  calcular Ekādaśī, se conservará el modelo propio.
- Si las diferencias pueden afectar el cálculo del tithi cerca de sus límites,
  se abrirá un ADR antes de rediseñar el modelo.
- Las pruebas no deberán inventar resultados de Swiss Ephemeris. Si el oráculo
  no está instalado localmente, la validación queda marcada como pendiente.

## Primera Validación Requerida

Antes de comenzar el modelo lunar se deberá ejecutar una comparación empírica
del Solar Engine v1.0:

```text
Pancanga Engine solar::apparent_longitude
contra
Swiss Ephemeris longitud aparente del Sol
```

Rango inicial:

```text
1900-2100
```

Muestra inicial:

```text
100 fechas astronómicamente sensibles
```

La muestra no será uniformemente distribuida. Deberá concentrarse en los casos
donde un error angular puede tener mayor impacto calendárico:

- 20 fechas aleatorias;
- 20 fechas cercanas a novilunio;
- 20 fechas cercanas a plenilunio;
- 20 fechas cercanas a cambios de tithi;
- 20 fechas extremas del rango de validación.

Métricas mínimas:

- error máximo;
- error medio;
- error RMS;
- fecha de mayor diferencia;
- unidades en grados, segundos de arco y, cuando sea útil, radianes.
- clasificación de la diferencia por tipo de muestra.

## Tolerancia

La tolerancia oficial queda:

```text
TBD
```

Se definirá después de comparar contra Swiss Ephemeris y evaluar el impacto
sobre el cálculo del tithi.

No se adopta todavía una tolerancia fija como `1"` porque la sensibilidad real
del cálculo completo todavía no fue medida.

## Riesgo Técnico Abierto

La comparación debe fijar de manera explícita:

- escala temporal usada por el `JulianDate`;
- flags de Swiss Ephemeris;
- si la longitud de referencia es tropical aparente;
- si la comparación usa TT o UT;
- normalización angular y manejo del cruce `0°/360°`.

Hasta que esos puntos estén cerrados, no se considerará científica ninguna
comparación numérica.

## Relación con ADR-0003

`ADR-0003` define la fuente de implementación astronómica inicial.

`ADR-0007` define la referencia externa de validación.

Estas decisiones no compiten entre sí.

## Referencias

- JPL Development Ephemerides, cuando corresponda.
- Jean Meeus, *Astronomical Algorithms*, 2nd Edition.
- IMCCE / Observatoire de Paris, VSOP87.

## Estado

Aceptado.

La medición empírica del Solar Engine v1.0 queda pendiente hasta disponer de
Swiss Ephemeris localmente.
