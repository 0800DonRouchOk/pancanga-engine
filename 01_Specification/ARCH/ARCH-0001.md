# Resumen

Este documento define la arquitectura oficial de Pancanga Engine.

Establece las capas, dependencias, principios y reglas que deberán respetar todos los componentes del proyecto.

ARCH-0001 constituye la referencia arquitectónica del motor y prevalece sobre cualquier decisión de implementación.

Toda modificación futura deberá documentarse mediante un ADR (Architecture Decision Record).

# 1. Propósito

Pancanga Engine es un motor de cálculo del calendario Gauḍīya Vaiṣṇava.

No es una aplicación.

No es una interfaz gráfica.

No depende de Internet.

Su única responsabilidad es producir resultados correctos y reproducibles.

---

# 2. Alcance

El motor será responsable del cálculo de:

- Tiempo Juliano
- Posición del Sol
- Posición de la Luna
- Nutación
- Oblicuidad
- Tiempo sidéreo
- Ayanāṁśa
- Amanecer y atardecer
- Tithi
- Nakṣatra
- Yoga
- Karaṇa
- Māsa
- Pakṣa
- Año Gaurābda
- Ekādaśī
- Mahādvādaśī
- Pāraṇa
- Festivales Gauḍīya

---

# 3. Filosofía

El proyecto se rige por los siguientes principios:

- Offline First.
- Determinismo absoluto.
- Arquitectura modular.
- Funciones puras siempre que sea posible.
- Compatibilidad verificable.
- Documentación antes que implementación.

---

# 4. Principios de Diseño

## 4.1 Single Responsibility

Cada módulo resolverá un único problema.

## 4.2 Bajo acoplamiento

Los módulos deberán conocerse lo menos posible.

## 4.3 Alta cohesión

Toda funcionalidad relacionada permanecerá dentro del mismo módulo.

## 4.4 Inmutabilidad

Los objetos de dominio serán preferentemente inmutables.

## 4.5 Determinismo

Una misma entrada siempre producirá exactamente la misma salida.

# 6. Capas del Sistema

## Core

Responsabilidades:

- Matemática
- Tiempo Juliano
- Ángulos
- Fechas
- Coordenadas
- Utilidades

No conoce astronomía.

---

## Astronomy

Responsabilidades:

- Sol
- Luna
- Nutación
- Oblicuidad
- Tiempo sidéreo
- Ayanāṁśa
- Amanecer
- Atardecer

No conoce Tithi.

---

## Lunisolar

Responsabilidades:

- Tithi
- Nakṣatra
- Yoga
- Karaṇa
- Pakṣa
- Māsa
- Gaurābda

No conoce Ekādaśī.

---

## Gaudiya

Responsabilidades:

- Ekādaśī
- Mahādvādaśī
- Pāraṇa
- Festivales
- Reglas Vaiṣṇavas

Nunca calcula posiciones astronómicas.

---

## Engine

Responsabilidades:

Coordinar el cálculo completo.

Entrada:

- Fecha
- Hora
- Ubicación
- Configuración

Salida:

DayContext

---

## Public API

Expone una interfaz sencilla para aplicaciones externas.

Ejemplos:

```rust
calendar.today()

calendar.compute(...)

calendar.next_ekadasi()

calendar.current_tithi()

calendar.current_naksatra()

calendar.parana()
```

La API nunca expondrá detalles internos de implementación.

# 8. Objetos Principales

## EngineConfig

Configuración global del motor.

Incluye:

- modelo solar
- modelo lunar
- ayanāṁśa
- idioma
- reglas Gauḍīya

---

## DayContext

Representa toda la información de un día.

Debe ser inmutable.

---

## JulianDate

Representación temporal interna.

---

## GeoLocation

Representación geográfica.

# 10. Recursos

Los recursos externos (nombres, idiomas, festivales, zonas horarias) estarán desacoplados del código y cargados desde archivos de datos.

---

# 11. API Pública

La API deberá ser:

- simple;
- estable;
- documentada;
- independiente de la implementación interna.

---

# 12. Testing

Cada módulo tendrá:

- pruebas unitarias;
- pruebas de integración;
- pruebas de regresión;
- comparación con PureBhakti;
- comparación con GCal.

Ningún módulo se considerará terminado sin superar su batería de pruebas.

---

# 13. Rendimiento

El motor deberá generar calendarios completos de manera eficiente en hardware de consumo.

La optimización nunca tendrá prioridad sobre la corrección.

---

# 14. Extensibilidad

La arquitectura permitirá incorporar:

- nuevos ayanāṁśas;
- nuevos idiomas;
- nuevos modelos astronómicos;
- nuevas reglas litúrgicas;
- nuevas interfaces.

Sin modificar el núcleo del sistema.

---

# 15. Compatibilidad

El perfil de configuración por defecto deberá reproducir los resultados del calendario PureBhakti para los casos validados.

Cuando existan alternativas de cálculo, deberán documentarse y ofrecerse mediante configuración, preservando el comportamiento por defecto.

---

# 16. Visión a Largo Plazo

Pancanga Engine aspira a convertirse en una biblioteca abierta, documentada y verificable para el cálculo del calendario Gauḍīya Vaiṣṇava.

El motor deberá permanecer estable durante muchos años y servir como núcleo para aplicaciones móviles, de escritorio, web, APIs y cualquier otra plataforma futura.
